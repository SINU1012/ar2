# Site Preview AR Overlay

모바일 브라우저에서 바닥/테이블을 인식해 `videoar.mp4` 영상을 3D 평면에 재생하는 WebXR AR 데모입니다. Android 크롬에서 WebXR AR을 지원하며, HTTPS 환경에서 동작합니다. (iOS Safari WebXR 미지원 시에는 안내만 표시)

## 실행 방법
1) 프로젝트 루트에서 정적 서버 실행  
   - `python3 -m http.server 8000`  
   - 또는 `npx serve .`  
2) 휴대폰 크롬에서 `https://<호스트>:8000` 혹은 배포 URL 접속 (로컬은 `https://<LAN IP>:8000` 권장).  
3) 페이지에 뜨는 시작 버튼을 누르면 카메라 권한을 요청하고 AR 세션을 시작합니다. (자동으로 막히면 탭 한 번 더)  
4) 영상은 카메라 정면 약 1m 앞(기본 너비 약 0.75m)으로 바로 나타나며, 한 손가락 드래그로 이동, 두 손가락 핀치로 크기 조정(0.3~2.5m)할 수 있습니다. 영상과 함께 따라오는 “사이트로 이동” 버튼을 눌러 `http://www.infocus1.co.kr`(80 포트) 로 접속할 수 있으며, 홀로그램 느낌의 글로우가 표시됩니다.

## 특징
- Three.js + WebXR로 카메라 정면에 비디오 텍스처 배치, 드래그/핀치로 이동·스케일, 영상 옆 “사이트로 이동” 버튼으로 사이트 연결
- 영상 비율 유지(기본 너비 약 0.9~1.0m, 높이 자동 비례, 메타데이터 없을 경우 기존 이미지 비율 1000x545 사용)
- 지원되지 않는 브라우저에서는 안내 문구로 대체

## 주의사항
- WebXR AR은 HTTPS + Android 크롬 최신 버전에서 동작합니다.  
- iOS Safari는 WebXR이 없어 AR 버튼이 표시되지 않을 수 있습니다(미래 버전에서 지원 시 자동 활성화).  
- GH Pages/Netlify 등 HTTPS 호스팅으로 배포하면 QR 코드로 스캔해 바로 테스트할 수 있습니다.
- AR 시작 버튼이 반응하지 않으면 HTTPS 접속 여부와 크롬에서 `immersive-ar` 지원 상태를 확인하세요. (지원 안 되면 상단에 안내 문구가 표시됨)
- 카메라 권한 요청이 보이지 않을 경우: 버튼을 탭하면 먼저 `getUserMedia`로 카메라 권한을 요청합니다. 거부되면 브라우저 설정에서 카메라 허용 후 다시 시도해야 합니다.
